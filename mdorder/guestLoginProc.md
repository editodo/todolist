# 요청 사항 (Requirements)

## 목표 (Goal)
- 체험계정 로그인 처리 와 본계정으로 변환 로직 구현

## 세부 요구사항 (Detailed Requirements)
1. [ ] 게스트가 체험버튼으로 로그인 했을 시 현재 임시계정으로 로그인 처리
2. [ ] 체험계정은 1개만 존재하도록
3. [ ] 체험계정은 7일 체험기간으로 설정
4. [ ] 체험기간 만료시 로그인 불가하지만 본계정으로 전환가능하도록 처리
5. [ ] 본계정 전환시 체험계정은 승계되어야함
6. [ ] 본계정 전환은 구글 로그인으로 가능하나 회원정보를 입력하면 전환해주는 방식으로 구현
7. [ ] 체험계정은 모든 기능을 사용가능하게 하되 로그인아이디가 없다고 하는 오류 및 에러는 되도록 나지않게 작업
8. [ ] 체험계정이 본계정으로 전환시 아이디가 변경될 수 있으니 체험계정과 신규로 만들어지는 아이디 간의 연동이 끊어지지않게 작업해서 전환된 본 계정으로 로그인을 하면 체험계정때의 데이터가 그대로 유지되도록 작업하되 체험계정 아이디는 로그인 못하게 해야함 삭제처리하거나 사용불가처리 
9. [ ] 체험계정으로 로그인시 체험계정임을 알 수 있게 해야함 
10. [ ] 마이페이지에 체험계정 및 본계정 아이디 닉네임 연동 표시 
11. [ ] 체험계정으로 로그인시 임의의 닉네임을 부여하고 컬럼 nickname 에 저장 
12. [ ] 닉네임과 로그인 아이디를 어떻게 처리할 것인지 고민 
13. [ ] 체험계정도 본계정도 STEP01~04까지 설정해놓은 데이터 값 저장 및 유지하고 마이페이지에서 테마 및 색상변경에서 변경한 값도 저장 및 유지되도록 작업해서 변경한 작업을 업데이트 시키는 로직 구현
